# _kano/backlog/

Local-first project backlog (file-based). Use this before code changes.

## Structure

- `_meta/`: schema and conventions
- `items/epic/`
- `items/feature/`
- `items/userstory/`
- `items/task/`
- `items/bug/`
- `decisions/`: ADRs
- `views/`: Obsidian Dataview dashboards

## Rules (summary)

- Create/update items before coding; meet the Ready gate.
- Worklog is append-only.
- Hierarchy is via frontmatter parent links, not folders.
